Ad Hoc DB Query Scripts

A small, self-contained toolbox for running ad hoc queries against the live pip-cost-sharing Firestore database and Firebase Auth. Uses the same firebase-admin SDK as the Cloud Functions, so it bypasses Firestore security rules and can query anything. Scripts run directly on Node 24 via built-in TypeScript type-stripping — no compilation step required.

⚠ These scripts target live production data. Keep queries read-only unless you are intentionally mutating documents.

Folder structure

scripts/db/
  lib.ts          — shared Firebase Admin SDK init + helper functions
  run.ts          — runner: resolves query name → queries/ file
  queries/        — your saved queries (add new .ts files here)
  package.json
  tsconfig.json
  README.html

One-time setup

  1. Install Google Cloud CLI if not already present.
  2. Log in with Application Default Credentials — this is what the Admin SDK uses to authenticate: 
    gcloud auth application-default login
    A browser window opens; sign in as the GCP project owner. Credentials are stored locally at ~/.config/gcloud/application_default_credentials.json — no service-account key file is created or committed.
  3. Install dependencies (run from the repo root): 
    pnpm install
    scripts/db is a pnpm workspace package, so this installs its dependencies (firebase-admin) alongside the main app.

Running queries

All commands run from the repo root.

Run a saved query

Pass the filename from queries/ without the .ts extension: 

pnpm query active-groups

This invokes run.ts, which looks up queries/active-groups.ts and executes it. Adding .ts explicitly also works: 

pnpm query active-groups.ts

Add a new query

Create a .ts file in scripts/db/queries/. Import helpers from ../lib.ts and write top-level await freely. No registration or config change needed — just run it by name. 

// scripts/db/queries/my-query.ts
import { db, logCount } from '../lib.ts';

const snap = await db.collection('users').count().get();
logCount('Total users', snap.data().count);
pnpm query my-query

Reference examples

The queries/ folder contains ready-made patterns but is not wired through the query runner. Run examples directly: 

node --experimental-strip-types scripts/db/queries/users-last-7-days.ts
node --experimental-strip-types scripts/db/queries/expenses-count-aggregation.ts
node --experimental-strip-types scripts/db/queries/groups-and-members.ts

Available helpers (lib.ts)

ExportDescription
db Firestore Admin SDK instance — use exactly like db in functions/src/index.ts
auth Firebase Auth Admin SDK — access user records, listUsers(), getUser(uid), etc.
dump(snap) Pretty-print every document in a QuerySnapshot
logCount(label, n) Print a labelled count: Total users: 42

Quick-reference patterns

import { db, auth, dump, logCount } from '../lib.ts';

// Count all documents in a collection
const snap = await db.collection('users').count().get();
logCount('Total users', snap.data().count);

// Count with a date filter (date strings are lexicographically comparable)
const thirtyDaysAgo = new Date(Date.now() - 30*864e5).toISOString().slice(0,10);
const recent = await db.collectionGroup('expenses')
  .where('date', '>=', thirtyDaysAgo).count().get();
logCount('Expenses in last 30 days', recent.data().count);

// Most recent expense in a group
const latestSnap = await db
  .collection('groups').doc('GROUP_ID')
  .collection('expenses').orderBy('date', 'desc').limit(1).get();
console.log(latestSnap.docs[0]?.data()['date']);

// Query across all groups using collectionGroup
const allMembers = await db.collectionGroup('members')
  .where('active', '==', true).get();

// Fetch docs and print them
const usersSnap = await db.collection('users').limit(10).get();
dump(usersSnap);

// Follow a DocumentReference field
const groupDoc = await db.collection('groups').doc('GROUP_ID').get();
const catRef = groupDoc.data()?.defaultCategoryRef;
if (catRef) {
  const catDoc = await catRef.get();
  console.log(catDoc.data());
}

// Firebase Auth — registration time (Firestore users have no createdDate)
const cutoff = Date.now() - 7*864e5;
let count = 0, token: string | undefined;
do {
  const page = await auth.listUsers(1000, token);
  count += page.users.filter(u =>
    new Date(u.metadata.creationTime).getTime() >= cutoff
  ).length;
  token = page.pageToken;
} while (token);
console.log(`New users in last 7 days: ${count}`);

Notes

  • Import from lib.ts using the .ts extension: import { db } from '../lib.ts'. This is required because scripts run via Node's type-stripping mode (no compilation).
  • The users collection has no createdDate field. Use Firebase Auth (auth.listUsers()) to filter by registration time.
  • Admin SDK bypasses Firestore security rules — handle with care.
