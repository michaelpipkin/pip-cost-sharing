Prerequisites

  • Android Studio installed, with the Android SDK Platform matching compileSdkVersion in android/variables.gradle (currently 36) installed via SDK Manager.
  • The release keystore file and its store password, key alias, and key password. This is not checked into the repo — it must exist somewhere safe on your machine (or wherever you normally keep it). Without it you cannot produce a bundle that Play will accept as an update, unless the app is enrolled in Play App Signing and you're using a separate upload key.
  • Access to the Google Play Console with permission to create releases for PipSplit.
  • Project dependencies installed (pnpm install — hand this off to a terminal you control if it hasn't been run recently).

Release steps

1

Bump the version numbers

Edit android/app/build.gradle — bump both fields:

versionCode 23        // must strictly increase from the last uploaded value
versionName "1.2.0"   // user-facing string, e.g. bump to "1.2.1" or "1.3.0"
versionCode must be higher than the highest value ever uploaded to any track (Production, Closed, Internal), not just the last Production release. Play rejects an upload that reuses or lowers it.

versionName is just a display string — bump it however fits (patch for a fix, minor for features).

If you also want the web/PWA version in sync, you can bump package.json's version field too, but it's independent of the Android version — pnpm run build:prod auto-bumps it as a patch version when building the web app, and it isn't read by the Android build.

2

Build the production web bundle and sync Capacitor

From the repo root:

pnpm run build:cap

This runs ng build --configuration=production, then npx cap sync, which copies dist/browser into android/app/src/main/assets/public and regenerates the native plugin config that Capacitor reads at runtime.

3

Open the project in Android Studio

npx cap open android

Let Gradle sync finish. If it complains about a missing SDK Platform, open Tools > SDK Manager and install the platform matching compileSdkVersion/targetSdkVersion in android/variables.gradle.

4

Smoke test on a device or emulator

Run the app (▶) before packaging a release. Confirm:

  • The app loads content from https://pipsplit.com correctly (per capacitor.config.ts's server.url).
  • Google Sign-In (Firebase Authentication) still works.
  • No crashes in Logcat — pay special attention right after bumping targetSdkVersion, since Android tightens behavior each release (edge-to-edge enforcement, predictive back gestures, notification permissions, etc.).
5

Generate the signed release bundle (AAB)

  1. Build > Generate Signed Bundle / APK…
  2. Choose Android App Bundle → Next.
  3. Point Key store path at your release keystore file. Enter the store password, key alias, and key password.
  4. Destination folder: default (android/app/release/) is fine.
  5. Build Variant: release.
  6. Click Create/Finish and wait for the build to finish.

Android Studio shows a notification with a "locate" link when done. The output file is android/app/release/app-release.aab.

6

Upload to Google Play Console

  1. Go to Play Console > PipSplit > Release, and pick a track (Production, or a testing track first if you stage rollouts).
  2. Create new release, upload app-release.aab.
  3. Write release notes ("What's new").
  4. Review — Play Console flags issues like target API level warnings here; this run's warning should be resolved now that targetSdkVersion is bumped.
  5. Save → Review release → Start rollout.
7

Commit the version bump

git add android/app/build.gradle android/variables.gradle
git commit -m "chore: bump Android release to 1.2.0 (versionCode 23)"

Push through your normal branch/PR workflow so the bumped version is tracked alongside the release.

Troubleshooting

"Version code already used" on upload
You reused or lowered versionCode. Check the highest code ever uploaded across all tracks in Play Console, then set a value higher than that.
Upload rejected for signature mismatch
You must sign with the same key used for all prior releases (unless the app is enrolled in Play App Signing with a separate upload key, in which case use that upload key). Using the wrong keystore/alias produces this error.
Gradle sync fails after bumping SDK versions
Open Tools > SDK Manager in Android Studio and install the missing Android SDK Platform/Build-Tools version, then re-sync.
Target API level warning persists in Play Console after upload
Confirm the uploaded .aab was actually built after the targetSdkVersion bump — a stale cached build in android/app/release/ is a common culprit. Rebuild the signed bundle.
